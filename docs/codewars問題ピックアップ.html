Triple Trouble

配列を使わない問題
3つ文字列を渡されるので、各文字列の先頭から順に合体させた文字列を返す。
前提として、入力される文字列はすべて同じ長さである。

例

"aa","bb","cc" --> "abcabc"
"Bm","aa","tn" --> "Batman"
"Cln","hlg","aee" --> "Challenge"

Copy!
public static String tripleTrouble(String one, String two, String three) {
	StringBuilder trouble = new StringBuilder();	// StringBuilderはStringより色々できる文字列
	for (int i = 0; i < one.length(); i++) {
		trouble.append(one.charAt(i));
		trouble.append(two.charAt(i));
		trouble.append(three.charAt(i));
	}


	return trouble.toString();
}

Copy!

以下反転で解説

String.charAt(int index)は文字列のindex番目の文字をcharで返すメソッド。
StringBuilder.append(char c)で文字列の末尾にcを追加するメソッド。
つまり、oneの文字列長の回数、各引数のi番目の文字をtroubleの末尾に追加している。

  • String (Java Platform SE 8 )

  • StringBuilder (Java Platform SE 8 )

Cat years, Dog years

人間の年齢(humanYears)を渡されるので、人間の年齢、猫の年齢(catYears)、犬の年齢(dogYears)を計算して返す。
条件は以下の通り。

  1. humanYears

    1. 必ず1以上である

    2. 必ず整数である

  2. catYears

    1. 最初の年は+15年

    2. 2年目は+9年

    3. その後は毎年+4年

  3. dogYears

    1. 最初の年は+15年

    2. 2年目は+9年

    3. その後は毎年+5年

例

 1 --> humanYears=1, catYears=15, dogYears=15
 2 --> humanYears=2, catYears=24, dogYears=24
10 --> humanYears=10, catYears=56, dogYears=64

Copy!
public static int[] humanYearsCatYearsDogYears(final int humanYears) {
	int catYears = 15;
	int dogYears = 15;

	if (humanYears > 1) {
		catYears += 9;
		dogYears += 9;
	}

	for (int i = 3; i <= humanYears; i++) {
		catYears += 4;
		dogYears += 5;
	}

	return new int[] { humanYears, catYears, dogYears };
}

Copy!

以下反転で解説

catYearsとdogYearsは1年目が15のため、初期値に15を設定して変数作成。
humanYearsが1より大きい場合はcatYearsとdogYearsに+9。（2年目の計算）
forで行う処理は「3年目以降の計算」のため、初期値を3とし、条件をhumanYears以下とする。

A wolf in sheep's clothing

あなたは羊飼いで、ヒツジのふりをするオオカミに悩まされています。
オオカミの前にいるヒツジに、オオカミに食べられることを警告します。
あなたは配列の最後にあるキューの先頭に立っていることを思い出してください。

["sheep","sheep","sheep","sheep","sheep","wolf","sheep","sheep"]
    7       6       5       4       3              2       1   

Copy!

オオカミがあなたに最も近い動物である場合は、「Pls go away and stop eat my sheep」を返します。
それ以外の場合は、「Oi! Sheep number N! You are about to be eaten by a wolf!」を返します。ここで、N はキュー内の羊の位置です。

配列には常に1つのオオカミが存在します。

例

Input: ["sheep", "sheep", "sheep", "wolf", "sheep"]
Output: "Oi! Sheep number 1! You are about to be eaten by a wolf!"

Input: ["sheep", "sheep", "wolf"]
Output: "Pls go away and stop eating my sheep"

Copy!
public static String warnTheSheep(String[] array) {
	if (array[array.length - 1].equals("wolf")) {
		return "Pls go away and stop eating my sheep";
	} else {
		for (int i = 0; i < array.length; i++) {
			if (array[i].equals("wolf")) {
				return "Oi! Sheep number " + (array.length - i - 1) + "! You are about to be eaten by a wolf!";
			}
		}
	}
	return "";
}

Copy!

以下反転で解説

やりたいことを整理すると、

  1. 配列の一番右にwolfが入っている場合は"Pls go away and stop eating my sheep"

  2. それ以外の場合はwolfの次にいる羊に"Oi! Sheep number N! You are about to be eaten by a wolf!"
    ただしNはwolfの次の番号

ここで、番号が右から割り振られていることに注意する。
つまり、配列の1番右の値を取得するときはarray.length - 1を指定しなければならない。

それらを考慮したうえで、まず最初に一番右の値がwolfか確認し、wolfではない場合にforで左からwolfかどうか確かめていく。
最初に一番右の値がwolfか確認をするのは、無駄な処理を省くため。（forのarray[i].equals("wolf")の中でif(i == array.length - 1)とするよりも効率的）

Gravity Flip

ボブは物理の授業が退屈なので、暇つぶしにおもちゃ箱を自作しました。この箱は、重力を変化させることができる特別なものです。

箱の中には立方体のおもちゃが何列か並んでいます。i番目の列にはa_i個のキューブが入っている。
最初、箱の中の重力はキューブを下に引っ張っている。ボブが重力を切り替えると、すべての立方体が箱のある辺dに引っ張られるようになる。

以下は、重力の切り替え前と切り替え後の立方体の箱がどのように見えるかの例である。

+---+                                       +---+
|   |                                       |   |
+---+                                       +---+
+---++---+     +---+              +---++---++---+
|   ||   |     |   |   -->        |   ||   ||   |
+---++---+     +---+              +---++---++---+
+---++---++---++---+         +---++---++---++---+
|   ||   ||   ||   |         |   ||   ||   ||   |
+---++---++---++---+         +---++---++---++---+

Copy!

箱の中の立方体の初期配置が与えられたとき、ボブが重力を切り替えた後に、n 個の列のそれぞれにある立方体の数を求めよ。
例

* 'R', [3, 2, 1, 2]      ->  [1, 2, 2, 3]
* 'L', [1, 4, 5, 3, 5 ]  ->  [5, 5, 4, 3, 1]

Copy!
// 配列のソートをしたいのでインポート
import java.util.Arrays;

public static int[] flip(char dir, int[] arr) {
	Arrays.sort(arr);

	// 逆順に並び替える処理
	if ('L' == dir) {	
		for (int f = 0, l = arr.length - 1; f < l; f++, l--) {
			int temp = arr[f];
			arr[f] = arr[l];
			arr[l] = temp;
		}
	}
	return arr;
}

Copy!

以下反転で解説

難しく考えず、Rは「右に傾けたら」、Lは「左に傾けたら」をイメージするとわかりやすい。
そのうえでやりたいことをよく考えると、配列のソートとわかるので、Rなら昇順、Lなら降順に並び替える。

ただし、Arrays.sort()は昇順にしかソートできないので、降順にしたいときは昇順にしてから改めて順を並び替える。
また、charはStringと違って==で比較できる。charはプリミティブ型でStringは参照型のため。詳細は各自で調べてください

Highest and Lowest

数字がスペースで区切られている文字列が与えられるので、その中の最大値と最小値を「スペースで区切った文字列」で返す。

条件は以下の通り。

  • 入力文字列には常に少なくとも 1 つの数値が含まれます。

  • 出力文字列は、1つのスペースで区切られた2つの数値で、最大の数値が最初。

例

Input: "1 2 3 4 5"
Output: "5 1"

Input: "1 2 -3 4 5"
Output: "5 -3"

Input: "1 9 3 4 -5"
Output: "9 -5"

Copy!
public static String highAndLow(String numbers) {
	String[] splitNumbers = numbers.split(" ");

	int max = Integer.parseInt(splitNumbers[0]);
	int min = Integer.parseInt(splitNumbers[0]);

	for (String numStr : splitNumbers) {
		int num = Integer.parseInt(numStr);

		if (max < num) {
			max = num;
		}

		if (min > num) {
			min = num;
		}
	}

	return max + " " + min;
}

Copy!

以下反転で解説

String.split(String regex)は、文字列をregexで分割した結果をString型の配列として返してくれる。
なので、まずは空白で分割した結果の配列を用意して、ひとつめの値をmaxとminに代入する。
その後forでひとつずつ取り出して比較それぞれと比較する。

Get the Middle Character

渡される文字列の中間の文字を返す。ただし、文字列の長さが偶数の場合は中間の2文字を返す。

例

"test" --> "es"
"testing" --> "t"
"A" --> "A"

Copy!
public static String getMiddle(String word) {
	int len = word.length();

	if (len == 1) {
		return word;
	}

	String result;
	boolean isOdd = len % 2 == 1;

	result = word.substring(len / 2 - 1, len / 2 + 1);

	if (isOdd) {
		result = result.substring(1);
	}

	return result;
}

Copy!

以下反転で解説

len % 2でlenを2で割ったときの余りが計算できるので、isOddにlen % 2が1か比較した結果を代入している。
String.substring(int beginIndex, int endIndex)は文字列をbeginIndexのindexからendIndexのindexまでを切り取って返してくれるので、
substring(len / 2 - 1, len / 2 + 1)で真ん中の2文字を取得する。
その後、もしwordの長さが偶数であれば、さっき取得した文字列の先頭をresult.substring(1)で取り除いている。
substringは引数が1つの場合、その引数のindex以降の文字を返す。

Reverse words

空白で単語が区切られている文字列を受け取り、単語を反転させた文字列を返す。
ただし、文字列内のスペースは全て保持する必要がある。
例

"This is an example!" --> "sihT si na !elpmaxe"
"double  spaces"      --> "elbuod  secaps"

Copy!
public static String reverseWords(final String original) {
	Pattern ps = Pattern.compile("\\S+");
	Pattern pw = Pattern.compile("\\s+");
	String[] spaces = ps.split(original);
	String[] words = pw.split(original);
	String result = "";

	switch (words.length) {
	case 0:
		return original;

	case 1:
		return new StringBuilder(original).reverse().toString();

	default:
		for (int i = 0; i < spaces.length; i++) {
			StringBuilder sb = new StringBuilder(words[i]);
			result += spaces[i] + sb.reverse();
		}
	}

	return result;
}

Copy!

以下反転で解説

Patternはコンパイル済みの正規表現。Pattern.compile()で作成できる。
Pattern.split(String)で文字列を設定した正規表現で分割する。
これを利用し、originalを1文字以上の非空白文字で分割した結果をspacesに、1文字以上の空白文字で分割した結果をwordsに代入する。
すると、

original:This is an example!
spaces:[""," "," "," "]
words:["This","is","an","example!"]

Copy!

という状態になる。
なぜspacesの先頭が空なのか？それは、1文字以上の非空白文字、ここでいう単語で区切っているため。
つまり、

[]This[ ]is[ ]an[ ]example!
↑     ↑    ↑    ↑

Copy!

の値を配列に代入しているイメージ。ただし、splitの結果として得られる配列には、後続の空の文字列は含まれない。

switchで、もし
wordsの長さが0、つまり空の場合、originalを返して終了する。
wordsの長さが1、つまり1つだけの場合、originalをreverse()で反転した値を返して終了。
それ以外の場合、繰り返し処理でresultにspacesのi番目の値とwordsのi番目の値を反転した文字列を結合していく。

Last Survivor

文字列と数字の配列が与えられます。

数字は、配列の先頭から順番に削除する必要がある文字の位置を示します。
削除するたびに、文字列のサイズが減少します (空のスペースはありません)。
最後に残された文字列を返します。

  • 指定された文字列が空になることはありません。

  • 配列の長さは、常に文字列の長さよりも 1 少なくなります。

  • すべての数値が有効です。

  • 文字と数字が重複している可能性があります。

例

str = "zbk", arr = {0, 1} --> b
str = "zbk", arr = {2, 1} --> z
str = "bk", arr = {1} --> b

Copy!
public static String lastSurvivor(String letters, int[] coords) {
	if (letters.length() == 1) {
		return letters;
	}

	StringBuilder sb = new StringBuilder();
	sb.append(letters);

	for (int i = 0; i < letters.length() - 1; i++) {
		sb.deleteCharAt(coords[i]);
	}

	return sb.toString();
}

Copy!

以下反転で解説

StringBuilder.deleteCharAt(int index)は、indexの位置にある文字を削除する。
これを利用し、forで繰り返しlettersの文字を削除している。

Red Knight

Red Knightは2つのポーンを追いかけています。どちらのポーンがRed Knightに捕まりますか？
入力は2つの整数になります。

  • knight: Red Knightの縦の位置。(0 or 1)

  • pawn: 2つのポーンの位置。(2 ~ 1000000)

出力はオブジェクトである必要があります。

public static class PawnDistance {
	private String color;
	private long distance;

	public PawnDistance(String s, long d) {
		color = s;
		distance = d;
	}
}

Copy!

  • color: どちらのポーンが捕まったか("Black" or "White")

  • distance: ポーンが捕まった位置。

条件は以下の通り。

  • Red Knightは常に水平位置0から開始します。

  • 黒のポーンは常に一番下にあります (垂直位置 1)。

  • 白いポーンは常に一番上 (垂直位置 0) にあります。

  • ポーンが先に動き、2つのポーンは同時に動きます。

  • Red Knightは前方に 2 マス、上下に 1 マス移動します。

  • ポーンは常に 1 マス前方に移動します。

  • 両方のポーンは同じ水平位置から開始します。

例

Input = 0, 4;
Output = new PawnDistance("White", 8);

Copy!
public static PawnDistance redKnight(int knight, long pawn) {
	String pawncolor;
	int x = (int) (pawn - knight);

	if (x % 2 == 0) {
		pawncolor = "White";
	} else {
		pawncolor = "Black";
	}

	return new PawnDistance(pawncolor, pawn * 2);
}

Copy!

以下反転で解説

一見ややこしいが、

  1. Red Knightは2歩、Pawnは1歩進むため、Pawnのスタート位置から倍の距離のところでPawnが捕まる

  2. Pawnのスタート位置が偶数の場合はRed Knightが最初にいた軸のPawnが、奇数の場合は最初にいた軸とは違う方のPawnが捕まる

ということに気づけばロジックは簡単。捕まる場所についてはpawn * 2で求められる。

どちらの色が捕まるかについては、単純に考えれば下記のようになる。

/**

if(knight == 0){
	// Pawnのスタート位置が偶数ならWhite、奇数ならBlack
	if(isEven){
		pawncolor = "White";
	}else{
		pawncolor = "Black";
	}
}else{
	// Pawnのスタート位置が偶数ならBlack、奇数ならWhite
	if(isEven){
		pawncolor = "White";
	}else{
		pawncolor = "Black";
	}
}

Status Arrays

明らかに7kyuの難易度ではないので注意！(6kyu以上の難易度?)

整数の配列の各要素のステータスは、配列内の位置と配列内の他の要素の値によって決定できます。
配列内の要素Nのステータスは、配列内の要素Nの位置を、配列内のN未満の配列要素の数に加算することによって決定されます。
たとえば、整数6,9,3,8,2,3,1を含む配列を考えてみます。要素8のステータスは8です。
これは、その位置が3であり、配列内に8未満の要素が5つあるためです。

元の配列の要素をステータスの低い順序から高い順序で返します。 2つ以上の要素の状態が同点の場合は、配列に出現する順に出力します。

例

Input = [6, 9, 3, 8, 2, 3, 1]
Output = [6, 3, 2, 1, 9, 3, 8] (ステータスが順に4, 4, 5, 6, 7, 7, 8)

Copy!
public static int[] status(int[] nums) {
	int x[] = new int[nums.length];
	for (int i = 0; i < x.length; i++) {
		int res = 0;
		for (int j = 0; j < x.length; j++) {
			if (nums[i] > nums[j]){
				res++;
			}
		}
		x[i] = i + res;
	}

	int y[] = new int[x.length];
	for (int i = y.length - 1; i >= 0; i--) {
		int index = 0;
		for (int j = 1; j < x.length; j++) {
			if (x[j] >= x[index]){
				index = j;
			}
		}
		y[i] = nums[index];

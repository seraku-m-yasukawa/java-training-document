質問されたこと 

Q：abstractとinterfaceの違いは？（やっていることがほとんど同じでは？）

A：イメージとしては、abstractを継承するときは、1つのカテゴリを表している。
　 例えば、クライアントしかできないこと、クライアントなら全員出来ることならabstract。
　 一方、interfaceは、クラス問わずにできること。
　 例えば、乗り物があるとする。「動く」ってやってしまえばできるが、「動く」って動作を乗り物以外、人間も動物も動くよねって時など、
　 全部に実装したいときに「動く」をinterfaceにする。

Q：インターフェイスは複数できる？
A：できます、カンマ区切りで記述してください

困りごとなど

  • 説明を聞いてその場で疑問に思ったことしか質問してこない

    • 考え方とかはあまり質問してくれない
      • たぶんそこまで追いついていない
  • わからなくてもヘルプをだしてこない、そのまま大丈夫って言っちゃう
    • わからなくて怒られるってプログラマだと（あんまり）ない気がするけど、他の業種だとそうでもないのだろうか……
      • 「わからなくて当たり前、ミスって当たり前、なのでそこで助けを求めてほしい」という前提を理解してもらわないと

質問されたこと 

Q：オブジェクトとインスタンスの認識に関して、オブジェクトとインスタンスは同様のものか、それとも相似的なものなのか？
　 あるサイトではオブジェクト＝インスタンスである...と書かれていたり、似て非なるものであると書かれていてどちらが正しいのか判断できず混乱している。

A：どちらがといわれるとちょっと難しいです。ただ、「オブジェクト=インスタンス」説が一番私の考えに近いです。
　 オブジェクトとインスタンスの認識については、概念的なものなので、自分でうまく解釈するしかないと思います。
　 個人的には、オブジェクト指向の考え方では「すべてのものはオブジェクト」らしいので、
　 インスタンスが「クラスにそった具体的なオブジェクト」で、クラスが「オブジェクトの分類（を示すオブジェクト）」だと思っています。

　 しかし現場では、インスタンスって言う人もオブジェクトって言う人もいるので、そこは文脈を読んで判断するしかないと思います。
　 とはいえ、基本的にjavaの話では「オブジェクト=インスタンス」と考えて問題ありません。

　 ちなみに、このサイトのページも参考になると思います。ご参考まで。
　 オブジェクトとは｜「分かりそう」で「分からない」でも「分かった」気になれるIT用語辞典 (i-3-i.info)

Q：現在int Aという変数にメソッドを呼び出して結果を入れているが、
　 呼び出し先のメソッドでエラーをキャッチした場合、なぜか次以降の入力の値が反映されない。

A：呼び出しとメソッドへの代入のタイミングがわるかった。

コードレビュー

実際のコードと指摘事項を主に記述している。

自分を再帰的に呼び出してしまうメソッド 

//スキャナー(戻り値：整数型)
public int inputInteger() {
    int answer = 0;
    try {
        String tempAnswer = scan.next();
        answer = Integer.parseInt(tempAnswer); //取得した値を整数型に変換
    }catch (NumberFormatException e){
        System.out.println("エラー: 入力された値は数値に変換できません。数値で入力してください。");
        System.out.print("＞");
        answer = inputInteger();            // <- ここでもうひとつinputIntegerを実行してしまう
    }
    return answer;
}

上記の設計だと、数字以外が入力された場合、新しくinputIntegerを呼びだしてしまいます。
その新しく呼び出したinputIntegerでも、数字以外を入力するとまた新しく呼び出して……というのを繰り返すと、
いずれStackOverflowErrorという、(ざっくりというと)メモリ不足のエラーが発生してしまいます。

そのため、continueなどを使ってループさせるのが良いと思いました。（今回はキーボード入力なのであまり考えなくてもいいですが、うっかり無限ループしてしまうのはあるあるです。）

余談ですが、gitでコミットするときは、コミットメッセージにある程度修正内容などを書き込むとわかりやすくなります。
例えば今回の内容を簡単に書くと、add: inputIntegerに数値以外を入力したときの処理を追加みたいになると思います。

/** 本は1冊も登録されておりません */				// <-- javadocの書式で記述すると、マウスオーバー時に情報が表示されるようになります
final String INFO_MESSAGE01="本は1冊も登録されておりません"

/**
 * 本の詳細情報確認								// <-- これもjavadoc パラメータなども表示されます
 */
public void bookInfo() {
	if (bookShelf.isEmpty()) { 					// <-- isEmpty()で listが空かどうか判定できます
		System.out.println(INFO_MESSAGE01); 	// <-- 繰り返し使うメッセージは定数にしておくと修正が楽です
	} else {
		System.out.print("確認したい本のISBNコードを入力してください　⇒　");
		option = scanner.nextInt();

		// 本棚に追加されている本の数だけ繰り返し
		int counter = 0;
		for (Book book : bookShelf) { 			// <-- 本棚の中のbookをforで取り出したほうがコードが読みやすいです
			if (book.getCode() == option) {
				System.out.println("ISBNコード：" + book.getCode() + "の検索結果を表示します");
				book.info();
				counter++;
				break;
			}
		}
		
		if (counter == 0) {
			System.out.println("お探しのISBNコードは登録されておりませんでした");
		}
	}
}
